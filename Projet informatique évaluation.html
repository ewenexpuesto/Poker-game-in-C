Évaluation du projet

Vos notes et celle de votre groupe

Vous trouverez sur cette page, pour chaque lot et chaque tâche, la liste des éléments qui sont évalués par votre chargé(e) de projet. À chaque élément est associé une petite explication pour que vous puissiez bien comprendre ce qui est attendu, ainsi que le résultat de l'évaluation par votre chargé(e) de projet. Vous pourrez ainsi discuter avec lui ou elle de ce que vous pouvez améliorer dans votre projet. Soyez vigilants, pensez à venir voir cette page au moins une fois par séance et, si ce n'est pas fait, à inciter vigoureusement votre chargé(e) à vous noter quand vous avez rendu votre travail.

Il est important de comprendre que votre note comprend à la fois une note commune et une note individuelle. Chaque tâche est associée à un responsable que vous et votre groupe désignerez. La qualité du résultat du projet profite à tout le groupe, la qualité du code et de la gestion du projet de chaque tâche ne profite qu'au responsable de la tâche en question. Il est bien entendu autorisé d'aider les autres pour leurs tâches, mais vous ne serez récompensé pour cela qu'à travers la note commune. Aussi, si votre encadrant remarque que une tâche dont vous êtes responsable a été faite par d'autres personnes, il peut tout à fait ne pas vous attribuer de point pour ce travail.

Utilisation de git

Votre travail doit être rendu avec l'outil de versionnement git. Afin de faciliter la correction, chaque message de commit doit être sous la forme "Lot X Tache Y Description du travail effectué". Sans quoi le chargé de projet n’est pas tenu de faire un effort.

Utilisation de GanttProject

La répartition des tâches se fera avec l'outil GanttProject. Vous devez indiquer avec cet outil qui est chargé de faire quoi.

Autres outils

Certains outils sont facultatif (par exemple les commentaires avec Doxygen ou les tests avec CUNIT).

Barême

Un barème est précisé à la toute fin de cette page, il est suivi du calcul de votre note. Quelques indications très importantes sont rappelées ici :

  • Si vous n'avez pas été nommé responsable d'au moins 3 tâches de type Info, 5 tâches de type Math et 1 tâches de type Recherche, vous avez 0/20. Tout le monde doit participer au projet et toute personne cherchant à profiter de ses camarades n'est pas la bienvenue.
  • Vous êtes responsable d'une tâche si vous avez été noté responsable dans l'outil GanttProject et si vous avez commité la majorité de votre tâche sur le dépot git.
  • Certaines tâches sont plus difficiles que d'autres. C'est comme ça. Répartissez-vous le travail en fonction de vos compétences.
  • Votre code doit être commenté.
  • Un fichier c qui ne compile pas avec la commande gcc -Wall -Wextra -std=c99 -c fichier.c -o fichier.o ou qui manifestement n'a pas été testé (i.e. qui plante au démarage) se voit attribuer la note de 0. Plus exactement la tâche associée au code ne rapporte aucun point à son responsable. Par contre la personne est tout de même nommée responsable de la tâche et celle-ci compte dans le total des tâches à effectuer.
  • On ne vous demande pas d'être parfait, vous avez le droit à l'erreur. Vous devez effectuer le travail sérieusement, c'est tout.
  • Inversement, ce n'est pas parce que vous respectez ces 3 points que vous aurez plus de 10/20 au projet, c'est le minimum syndical, il ne veut pas dire que votre travail est bien fait. Sachez toutefois que la magnanimité de votre responsable d'UE n'a aucune limite. Un mauvais résultat peut s'expliquer de moult manières que le système d'évaluation ici présent ne peut capturer. Il est toujours possible de discuter et si vous avez travaillé sérieusement, vous aurez sans doute la moyenne au projet moyennant possiblement un petit travail pour remonter votre note.

Dernier point important : Aucun rapport ni aucune soutenance ne sont exigés par défaut pour ce projet, sauf si cela est explicitement demandé dans les évaluations ci-dessous. Toutefois, des points réguliers (à chaque séance) sont à faire avec vos chargés de projet. Vous devez être capable de dire où vous en êtes, d’indiquer les problèmes que vous rencontrez, ... La présence en séance est obligatoire. Une petite présentation de votre projet (sous forme de démonstration, de rapport ou de soutenance) peut être un plus et permet de donner un peu plus de crédit à votre travail, un sentiment d’accomplissement mais cela ne doit pas se faire au détriment de votre travail.

Améliorer votre note après correction, hors délai

Chaque lot est associé à une date de rendu. Cette date est indicative, mais la respecter rapporte quelques points à tout le groupe.

Lorsque votre lot est rendu à l'heure et a été évalué par votre encadrant(e), vous avez 3 possibilités :

  • Ne rien changer, votre note et votre lot ne changent pas.
  • Committer du code et demander une nouvelle correction, si la date de rendu est dépassée, vous perdez les points de rendu du lot à l'heure, mais vous améliorez possiblement la qualité de votre rendu. Les notes de votre lot et de vos tâches peuvent augmenter.
  • Committer du code et mais ne pas demander une nouvelle correction, la note de votre lot ne change pas, mais la qualité de votre lot augmente. Ceci peut être nécessaire pour démarrer un lot ultérieur sur une bonne base.
Pensez à en discuter avec votre chargé(e) de projet

Note individuelle

Toutes les tâches dont vous êtes responsable sont évaluées, vous attribuant pour chacune une note d'au plus 4.00 points en Info, 3.00 points en Math et 8.00 points en Recherche. Plus la tâche est bien évaluée par votre tuteur et plus grand est le nombre de points. Ne sont ensuite conservées que les meilleures tâches (3 tâches de type Info, 5 tâches de type Math et 1 tâches de type Recherche) dont les points sont additionnés pour obtenir votre note individuelle. On rappelle qu'une tâche qui ne compile pas ou qui n'a manifestement pas été testée se voit attribuée la note de 0.
